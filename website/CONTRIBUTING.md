# Contributing to the ProseKit website

This file is the content style guide for `website/src/content/docs/`. It
applies to every hand-written MDX page (extensions, components, concepts,
frameworks, guides). It does **not** apply to the auto-generated
`references/**` tree, which is regenerated by `build-typedoc.sh` from
JSDoc.

## One-sentence `description:` in every frontmatter

Every MDX page must carry a `description:` of ≤ 160 characters. Sentence
case, no trailing period.

```yaml
---
title: Bold
description: "Bold text formatting backed by the standard <strong> mark"
---
```

## Public API only

Never document `BaseExtension`, `Tuple5`, `Facet`, `FacetNode`, or any
file path under `packages/*/src/`. These are internal implementation
details. Refer to public symbols only, by their stable umbrella import
(`prosekit/...`).

## Use the umbrella package name in import statements

```ts
// bad
import { defineBold } from '@prosekit/extensions/bold'
// good
import { defineBold } from 'prosekit/extensions/bold'
```

The scoped `@prosekit/*` packages exist but are not the user-facing API.

## Code blocks

- `` ```ts twoslash `` for type-checked snippets.
- `` ```ts `` for snippets that twoslash can't process (e.g. component-only
  imports).
- `` ```bash `` for shell commands.

## Cross-links

- Inside `references/**` (auto-generated), use relative `*.md` paths
  (`rehype-resolve-markdown-links` rewrites them at build time).
- In hand-written pages, use absolute `/section/page` paths.
- When linking to a TypeDoc symbol, prefer the form
  `[`functionName`](/references/<module>#<anchor>)`.

## Live demos

Every page that introduces a feature must include
`<Demo story="…" />` for the canonical example. Story ids live in
`registry/src/stories.ts`.

## Keyboard symbols

Always use `<Keyboard key="…" />`, never raw text or `<kbd>`.

## Spelling

Use US English everywhere (`color`, `behavior`, `customize`, `serialize`).
Avoid UK spellings (`colour`, `behaviour`, `customise`, `serialise`).

## Frontmatter conventions

- `title`: title-case.
- `description`: sentence-case, no trailing period, ≤ 160 chars.
- `sidebar.label`: derived from `title` automatically; only set
  explicitly when the visible label needs to differ.
- Numeric `sidebar.order`: 10, 20, 30 … (room to insert).
