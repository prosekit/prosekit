The columns extension adds a block-level layout node that groups regular block
content into side-by-side columns. Each column remains normal editor content,
so paragraphs, headings, lists, images, and other block nodes continue to work
inside the layout.

<Demo story="columns" />

## Usage

```ts twoslash
import 'prosekit/extensions/columns/style.css'
import { defineColumns } from 'prosekit/extensions/columns'

const extension = defineColumns({
  enableModAWithinColumn: true,
  minColumnWidth: 160,
  defaultGap: 20,
})
```

## Behavior

- `insertColumns` replaces the current selection with a new columns container.
- `addColumnBefore` and `addColumnAfter` insert adjacent columns around the
  current one.
- `removeColumn` deletes the current column. When only one column would remain,
  the container is unwrapped back into regular block content.
- `distributeColumns` resets the current columns container to equal widths.
- `setColumnsGap` updates the gap between columns.

The extension ships with the node specs, commands, plugin state for resize UI,
and an optional `Mod-a` keymap that selects only the current column content.

## Styling

Import `prosekit/extensions/columns/style.css` to get the default layout styles.
The stylesheet exposes these CSS variables:

- `--prosekit-columns-gap`
- `--prosekit-column-width`

The built-in stylesheet only handles the rendered column layout. Resize handles
and drag affordances are application UI. The demo shows one way to build that
UI on top of the exported plugin state and helpers.
