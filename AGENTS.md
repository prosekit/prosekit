# Repository Guidelines

This file provides guidance to AI agents when working with code in this repository.

## Project Overview

This is a monorepo for `aria-ui`, a framework for building light-DOM-based accessible web components using reactive primitives. The project uses pnpm workspaces and consists of:

- `packages/core` - Core reactive framework with signals, contexts, and element base classes
- `packages/elements` - Web components built on `@aria-ui/core` (e.g., Menu components)
- `website` - Astro-based documentation site using Astrobook

## Package Manager

This project uses **pnpm**. Always use `pnpm` commands, never `npm` or `yarn`.

## Common Commands

### Development

```bash
# Start development server (runs website)
pnpm dev

# Run tests
pnpm test

# Type checking
pnpm typecheck

# Linting and formatting
pnpm lint
pnpm fix
```

### Package-specific commands

```bash
# Build individual packages
pnpm --filter @aria-ui/core build
pnpm --filter @aria-ui/elements build

# Run website build
pnpm --filter website build
```

## Architecture

### Core Framework (`packages/core`)

The core package provides a reactive component framework built on:

- **ReactiveElement**: Base class extending HTMLElement with reactive controller support
- **HostElement**: Component base class that combines reactive elements with props and setup functions
- **Signals**: Fine-grained reactivity using `alien-signals` library
- **Store**: Collection of signals representing component props, automatically synced with element properties
- **Context**: Event-based context system for providing/consuming values across the DOM tree
- **useEffect**: Hook for managing side effects with cleanup
- **useId**: Hook for generating unique IDs
- **useInteraction**: Hook for interaction handling (event listeners)

### Component Architecture

#### ComponentGroup vs ComponentPart

- **ComponentGroup**: A logical grouping of related web components (e.g., `Menu`, `Button`)
- **ComponentPart**: Individual web component within a component group (e.g., `FooRoot`, `FooItem` are component parts of the `Foo` component group)
- A ComponentGroup may have one or multiple ComponentParts. For example:
  - `Button` component group has 1 part: `Button`
  - `Menu` component group has at least 2 parts: `MenuRoot`, `MenuItem`

#### File Structure

For each **ComponentPart** (e.g., `FooRoot`), create:

- `packages/elements/src/foo/foo-root.ts` - ComponentPart implementation (contains Props, PropsDeclaration, Events, and setup function)

**Note:** The `{ComponentPartName}Element` class is automatically generated by `packages/cli/src/generate.ts` to `packages/elements/src/generated/elements/{component-name}.gen.ts`. Do NOT create manual `exports.ts` or `index.ts` files for ComponentGroups.

#### ComponentPart File Structure

Each ComponentPart file (e.g., `foo-root.ts`) must follow this exact order:

```typescript
/**
 * @public
 */
export interface FooRootProps {
  // Props definition

  /**
   * Whether the menu is initially open.
   *
   * @default false
   */
  defaultOpen: boolean
}

/**
 * @internal
 */
export const FooRootPropsDeclaration =
  /* @__PURE__ */ defineProps<FooRootProps>({
    // Props defaults

    defaultOpen: {
      default: false,
    },
  })

/**
 * @public
 * Optional: Only include if the component emits events
 */
export interface FooRootEvents {
  // Events definition

  /**
   * Emitted when the menu is opened or closed.
   */
  openChange: OpenChangeEvent
}

/**
 * @public
 */
export interface FooRootState {
  side: Side
}

/**
 * @internal
 */
export function setupFooRoot(host: HostElement, props: Store<FooRootProps>) {
  // Setup logic
}

// Other exports follow...
```

#### Props Requirements

**IMPORTANT:** Every property in `{ComponentPartName}Props` must follow these rules:

1. **Must have `@default` tag**: Every property must include a JSDoc `@default` tag documenting its default value
2. **Must NOT be optional**: Properties must not use the optional `?` modifier (e.g., use `prop: string`, not `prop?: string`)
3. **Default values must match**: The value specified in the `@default` JSDoc tag must exactly match the `default` value in the corresponding `{ComponentPartName}PropsDeclaration`

Example:

```typescript
export interface FooRootProps {
  /**
   * Whether the menu is initially open.
   *
   * @default false  // ← Must be present
   */
  defaultOpen: boolean // ← Must NOT be optional (no ?)
}

export const FooRootPropsDeclaration = defineProps<FooRootProps>({
  defaultOpen: {
    default: false, // ← Must match @default value above
  },
})
```

#### Export Visibility Rules

Every export must be marked with `@public` or `@internal`:

**Always `@public`:**

- `{ComponentPartName}Props` interface (if exists)
- `{ComponentPartName}Events` interface (if exists)

**Always `@internal`:**

- `{ComponentPartName}PropsDeclaration` constant (if exists)
- `setup{ComponentPartName}` function

**Context-dependent:**

- Other exports: `@public` if referenced by public API, otherwise `@internal`

#### Import Rules

**All imports:**

- Always import directly from ComponentPart files (e.g., `./foo-root.ts`)

  ```typescript
  import type { FooRootProps } from '../foo/foo-root.ts'
  import { setupFooRoot, FooRootPropsDeclaration } from '../foo/foo-root.ts'

  const props: FooRootProps = {}
  ```

#### Registration

All elements are registered in `packages/elements/src/register.ts` using the generated register functions:

```typescript
import { registerFooItemElement } from './generated/elements/foo-item.gen.ts'
import { registerFooRootElement } from './generated/elements/foo-root.gen.ts'

export function registerElements(): void {
  // Skip registration on non-browser environments
  if (typeof window === 'undefined') {
    return
  }

  registerFooRootElement()
  registerFooItemElement()
  // ... register all other elements
}
```

**Rules:**

- Import `register{ComponentPartName}Element` functions from generated `.gen.ts` files
- Call each register function to register the custom element
- The element tag name is automatically set to `aria-ui-{component-name}` by the generator
